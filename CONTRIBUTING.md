# Contributing

Thanks for helping improve SOC Alert Tracker.

## How to contribute

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make a focused change with a clear commit message.
4. Run the app locally and verify the changed workflow.
5. Open a pull request with a short summary and any testing notes.

## Good first areas

- Add automated tests for alert workflows and role permissions.
- Improve dashboard filters, charts, and reporting views.
- Add integrations for SIEM enrichment, notifications, and ticket export.
- Improve documentation for deployment, backups, and production configuration.
- Strengthen security defaults around secrets, authentication, and audit logging.

## Security

Do not commit real tokens, passwords, customer data, alert evidence, or local database files.

If you find a security issue, avoid posting sensitive details publicly. Open a minimal issue asking for a private disclosure path.
